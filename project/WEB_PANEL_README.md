# Web Panel

این پنل وب برای کار کردن کنار ربات تلگرام ساخته شده و از همان دیتابیس MySQL تعریف‌شده در `database.py` استفاده می‌کند.

## راه‌اندازی

1. وابستگی‌ها:

```powershell
python -m pip install -r requirements.txt
```

2. تنظیمات:

یک فایل به نام `web_panel_settings.json` کنار `web_panel.py` بسازید یا متغیر `WEB_PANEL_SETTINGS` را به مسیر فایل تنظیمات بدهید. قالب آماده در `web_panel_settings.example.json` قرار دارد.

پسورد ورود ادمین را در همین فایل با کلید `admin_web_password` تنظیم کنید. کاربران عادی پسورد وب خودشان را از داخل ربات با گزینه `تنظیم پسورد وب` ثبت می‌کنند.

3. اجرای پنل:

```powershell
python web_panel.py
```

آدرس پیش‌فرض:

```text
http://127.0.0.1:5050
```

## نکته مهم درباره همگام‌سازی

همه داده‌هایی که داخل MySQL هستند، بین ربات و پنل مشترک‌اند: کاربران، موجودی، کانفیگ‌ها، کانفیگ‌های سروری، سرورها و صف پرداخت.

ورود وب با `chat_id + password` انجام می‌شود. سطح دسترسی ادمین همچنان با chat_idهای تعریف‌شده برای ربات تعیین می‌شود، اما پسورد ورود ادمین جدا و ثابت در تنظیمات وب‌پنل است.

چند مقدار در کد فعلی ربات داخل حافظه `config.py` نگهداری می‌شوند، مثل لیست ادمین‌ها، کانال‌ها، قیمت‌ها و شماره کارت. پنل برای این‌ها فایل `web_panel_settings.json` دارد. برای همگام‌سازی کامل و زنده این تنظیمات، قدم بعدی این است که خود ربات هم همین تنظیمات را از دیتابیس یا همین فایل مشترک بخواند.

## جداسازی مسیر شبکه

ارتباط‌های تلگرام و ارتباط‌های 3x-ui عمداً از هم جدا شده‌اند:

- تمام ارتباط‌های تلگرام از مقدار `telegram_proxy_url` در `web_panel_settings.json` استفاده می‌کنند.
- تمام ارتباط‌های سرورهای 3x-ui با session مستقیم ساخته می‌شوند و `trust_env=False` دارند؛ بنابراین از `HTTP_PROXY` / `HTTPS_PROXY` سیستم یا پروکسی تلگرام اثر نمی‌گیرند.

نمونه مقدار برای پروکسی تلگرام:

```json
"telegram_proxy_url": "socks5h://127.0.0.1:9050"
```

برای پروکسی SOCKS، وابستگی `PySocks` هم در `requirements.txt` اضافه شده است.

## عملیات ادمین وب

پنل وب عملیات ادمین ربات را هم پوشش می‌دهد:

- روشن/خاموش کردن فروش و عملیات ساخت/تمدید با همان فایل `config.json`
- بکاپ گرفتن از همه سرورهای 3x-ui
- ارسال پیام همگانی تلگرام به کاربران
- مشاهده آمار ماهانه CSV از مسیر `/var/bot/stats`
- افزایش موجودی ادمین اصلی
- جایگزینی کامل سرور و انتقال لینک‌های subscriptionها به سرور جدید
