سرورها

ثبت سرور جدید

اتصال به پنل 3x-ui

آدرس پنل باید همان آدرسی باشد که API پنل روی آن لاگین می‌کند.

انتخاب inbound

اگر سرور inbound آماده داشته باشد، دکمه بررسی را بزنید و از لیست انتخاب کنید. اگر سرور خالی است، حالت ساخت inbound جدید را بگذارید.

برای اتصال بدون خطا به inbound موجود، تنظیمات از خود 3x-ui خوانده و ذخیره می‌شود.

نوع کانفیگ

برای حالت‌های ساده، همین گزینه‌ها کافی است. برای Reality، gRPC یا تنظیمات خاص، JSON را از 3x-ui کپی کنید.

TLS و certificate

اگر TLS ندارید، هر چهار مقدار را همان -1 بگذارید. اگر TLS دارید، مسیر certificate و key باید روی خود سرور 3x-ui معتبر باشد.

تنظیمات پیشرفته 3x-ui

برای Reality، gRPC، WebSocket host/path، SplitHTTP و templateهای خاص، JSONهای همین بخش را از inbound پنل 3x-ui کپی کنید.

جایگزینی سرور

انتخاب و مقصد جایگزینی

همه کانفیگ‌های روی سرور قبلی به سرور جدید منتقل می‌شوند و لینک کاربران به‌روزرسانی می‌شود.

نوع کانفیگ جدید

این تنظیمات باید با سرور مقصد سازگار باشند؛ در حالت‌های پیچیده، streamSettings را از 3x-ui کپی کنید.

TLS و certificate جدید

اگر مقصد TLS ندارد، این چهار فیلد را -1 بگذارید. برای Reality معمولاً باید streamSettings کامل را وارد کنید.
